# Weather App (Compose Multiplatform) — Lab 9

Кроссплатформенное приложение для отслеживания погоды в реальном времени, разработанное в рамках Лабораторной работы №9. Поддерживает **Android**, **iOS**, **Desktop (Linux/Windows/macOS)** и **Web (Wasm)**.

## 🚀 Функциональные возможности
- **Поиск погоды:** Ввод названия города и получение актуальных данных через OpenWeatherMap API (на русском языке).
- **Подробные данные:** Температура (°C), текстовое описание состояния неба, влажность и скорость ветра.
- **Кеширование:** Автоматическое сохранение последних полученных данных для просмотра в офлайн-режиме (используется Multiplatform Settings).
- **История поиска:** Быстрый доступ к последним 5 городам через интерактивные чипы.

## 🛠 Технологии
- **UI:** Compose Multiplatform (Material 3).
- **Network:** Ktor Client (OkHttp для Android/Desktop, Darwin для iOS).
- **Serialization:** kotlinx.serialization (JSON-парсинг ответов OpenWeatherMap).
- **Asynchrony:** Kotlin Coroutines.
- **Storage:** Multiplatform Settings (локальное хранилище данных и истории).
- **Images:** Coil 3 (асинхронная загрузка иконок погоды).

## 📱 Адаптация UI под платформы
Приложение активно использует механизм `expect/actual` для соответствия специфическим требованиям платформ:
- **Android:** Использование `ElevatedCard` с тенями и скруглениями. Стандартный `TextField` с иконкой поиска внутри.
- **iOS:** Плоский дизайн карточек (без теней), стилизованный под систему `SearchBar` и `SegmentedControl` для быстрого переключения между популярными городами.
- **Linux (Desktop):** Минималистичный дизайн с четкими границами (`border`) и акцентной цветовой полосой на карточках. Использование `OutlinedTextField` с прямоугольными формами.
- **Web / Desktop:** Отзывчивая верстка на основе `BoxWithConstraints`:
  - 1 колонка для мобильных устройств.
  - 2 колонки для планшетов.
  - 3 колонки для десктопных экранов и веб-интерфейса.

## 🧪 Тестирование
В проекте реализовано комплексное тестирование:
- **Unit-тесты:** Проверка логики парсинга сетевых моделей и обработки истории поиска.
- **Integration-тесты:** Проверка взаимодействия репозитория с мок-клиентом Ktor и хранилищем Settings.
- **UI-тесты:** Базовая проверка корректности отображения компонентов на различных платформах.

## 🏗 Сборка и запуск (CI/CD)
Проект содержит настроенный рабочий процесс **GitHub Actions** (`build.yml`):
- **iOS/macOS:** Сборка Klibs и Desktop-дистрибутива на macOS-latest.
- **Android/Web/Linux:** Сборка APK, Wasm-дистрибутива и JAR на ubuntu-latest.
